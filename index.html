1. Access Tokens

What are Access Tokens??

  • These are the authorization tokens.
  • They are the sign of authorized user.
  • These are a kind of barrier for hackers.

How These Works??

  • When the user is logged it is registered with an token with some expiring time limit.
  • And After login whenever request anything from server, user has to send that token with the request, so that server came to know that this request is requested form a valid user .
  • If the token doesn't matches then server sends an error response that you are not a valid user. And ya user Data is Safe.

This is how JWT Auth works

2. Password Hashing

press ↓ to know more

What does the term Hashing Means??

  • Firstly, Hashing is not an Encryption.
  • Hashing is converting a readable string into a large unreadable coded string and we never generate the actual string from this hash string now s.
  • Encryption is converting a readable string into an encrypted string but we can decrypt or re-generate the actual string

Do you know how your password being stored in database?

If it's stored as it is then developer can see your password and access your account.

So what's the solution??

How Does Password Hashing works??

  • As user we set-up our password its being converted into hash key and stored into the database.
  • So the developer can only see your hashed key not your actual password.
  • So how I'll do login next time if its stored as hashed string??
  • When the user comes again for login then also his password get's hashed.
  • And we use such hash algorithms that same hash string will generate again and we get authorized and logged-In.

Some Hash functions

  • Sha512
  • Sha256
  • md5
